# 下一条视频标准输入格式

以后每条视频先按这个格式准备。用户不需要理解代码，只需要提供内容和素材。

## 1. 主题

一句话说明这条视频讲什么。

示例：

```text
我用 AI 把一段乱想法整理成视频脚本。
```

## 2. 想表达的话

可以很乱，不需要成稿。ChatGPT 会整理成口播稿。

示例：

```text
我以前写东西很乱，想到哪写到哪。后来我发现可以先让 AI 问我几个问题，再帮我整理成结构。
```

## 3. 素材

优先提供截图，其次提供录屏。

推荐命名：

```text
01-xxx.png
02-xxx.png
03-xxx.png
```

素材类型：

- GitHub 页面截图
- ChatGPT 对话截图
- Codex 执行结果截图
- Remotion Studio 截图
- 输出文件夹截图
- 工具操作录屏

## 4. 配音

配音格式：

```text
voice-002.m4a
```

要求：

- 不加背景音乐
- 安静环境
- 正常语速
- 不需要一次完美，后续可以重新录

## 5. ChatGPT 输出内容

ChatGPT 负责输出：

1. 正式口播稿
2. 分镜结构
3. 素材对应表
4. JSON 配置
5. 给 Codex 的最小执行指令

## 6. Codex 执行内容

Codex 只负责：

1. 复制素材到项目目录
2. 拉取 GitHub 最新代码
3. 类型检查
4. 渲染视频
5. 报错时做最小修复

## 7. 固定原则

- 不让 Codex 重新设计。
- 不让 Codex 大改项目。
- 不把真人音频提交到公开仓库。
- 不把每条视频都从零做。
- 每次只替换本期内容和素材。
